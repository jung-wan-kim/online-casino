# 온라인 카지노 웹사이트 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트 목표
실제 카지노 게임 경험을 온라인으로 재현하는 고품질 웹 기반 카지노 플랫폼 개발

### 1.2 핵심 가치
- **실감나는 게임 경험**: 실제 카지노와 동일한 비주얼 및 게임 플레이
- **공정한 게임**: 검증 가능한 랜덤 알고리즘 사용
- **사용자 친화적 인터페이스**: 직관적이고 세련된 UI/UX
- **안전한 금융 거래**: 보안이 강화된 칩 교환 시스템

### 1.3 법적 고려사항
⚠️ **중요**: 온라인 도박은 국가별로 법적 규제가 다릅니다.
- 운영 전 해당 국가의 온라인 도박 관련 법규 확인 필수
- 게임 라이선스 취득 필요 (운영 지역에 따라)
- 미성년자 접근 차단 시스템 구현 필수
- 책임감 있는 게임(Responsible Gaming) 정책 수립
- 자금 세탁 방지(AML) 정책 준수

## 2. 핵심 게임 - 카드게임 3종

### 2.1 블랙잭 (Blackjack)
**게임 개요**: 딜러와 플레이어가 21에 가까운 숫자를 만드는 게임

**게임 규칙**:
- 카드 합이 21을 넘지 않으면서 딜러보다 높은 수를 만들기
- A는 1 또는 11, J/Q/K는 10, 나머지는 숫자 그대로
- 플레이어 액션: Hit(카드 받기), Stand(멈추기), Double Down(배팅 2배), Split(같은 카드 분리)
- 블랙잭(A + 10): 1.5배 배당

**구현 요구사항**:
- 6덱 또는 8덱 슈(shoe) 시스템
- 카드 카운팅 방지 메커니즘
- 실시간 카드 애니메이션
- 딜러 규칙: 16 이하 Hit, 17 이상 Stand
- 인슈어런스(Insurance) 옵션

### 2.2 바카라 (Baccarat)
**게임 개요**: 뱅커(Banker)와 플레이어(Player) 중 어느 쪽이 9에 가까운지 맞추는 게임

**게임 규칙**:
- 카드 합의 일의 자리 숫자로 승부 (9가 최고)
- A는 1, 10/J/Q/K는 0, 나머지는 숫자 그대로
- 베팅 옵션: Player, Banker, Tie
- 배당: Player 1:1, Banker 0.95:1 (5% 수수료), Tie 8:1
- 자동 드로우 규칙 적용 (3장째 카드)

**구현 요구사항**:
- 실시간 결과 통계 (Player/Banker/Tie 비율)
- 로드맵 (Big Road, Bead Road, Big Eye Road 등)
- 자동 슈플 시스템
- 페어 베팅 옵션 (같은 숫자 2장)

### 2.3 포커 (Texas Hold'em)
**게임 개요**: 커뮤니티 카드와 핸드 카드로 최고의 5장 조합을 만드는 게임

**게임 규칙**:
- 각 플레이어에게 2장의 핸드 카드
- 5장의 커뮤니티 카드 (Flop 3장, Turn 1장, River 1장)
- 베팅 라운드: Pre-Flop, Flop, Turn, River
- 액션: Fold, Check, Call, Raise, All-in
- 핸드 랭킹: Royal Flush > Straight Flush > Four of a Kind > Full House > Flush > Straight > Three of a Kind > Two Pair > One Pair > High Card

**구현 요구사항**:
- 멀티플레이어 테이블 (2-9명)
- 실시간 채팅 시스템
- 타이머 시스템 (액션 제한 시간)
- 팟(Pot) 관리 및 사이드팟 계산
- 자동 핸드 히스토리 저장

## 3. 핵심 시스템

### 3.1 칩 교환 시스템

**기능**:
- 실제 화폐 → 게임 칩 전환
- 게임 칩 → 실제 화폐 환전
- 다양한 칩 단위 (1, 5, 25, 100, 500, 1000, 5000)
- 결제 수단 통합 (카드, 계좌이체, 전자지갑)

**보안 요구사항**:
- PCI DSS 준수
- 2단계 인증(2FA)
- 거래 내역 암호화
- 실시간 사기 탐지 시스템
- 인출 한도 및 검증 프로세스

**UI/UX**:
- 캐시어(Cashier) 페이지
- 실시간 잔액 표시
- 거래 히스토리
- 입출금 상태 추적

### 3.2 딜러 시스템

**라이브 딜러 모드**:
- 실제 딜러와의 실시간 스트리밍
- HD 비디오 스트리밍 (WebRTC)
- 다중 카메라 앵글
- 채팅을 통한 딜러와의 상호작용

**AI 딜러 모드**:
- 3D 아바타 딜러
- 음성 안내 (TTS)
- 랜덤 동작 및 표정
- 24/7 무중단 운영

**딜러 관리**:
- 딜러 스케줄링 시스템
- 성과 모니터링
- 교육 및 인증 관리

### 3.3 게임 비주얼 시스템

**카드 렌더링**:
- 고해상도 카드 이미지 (4K)
- 부드러운 애니메이션 (60fps)
- 3D 카드 효과 (뒤집기, 슬라이딩)
- 물리 기반 렌더링

**테이블 디자인**:
- 사실적인 펠트 질감
- 동적 조명 효과
- 그림자 및 반사
- 커스터마이징 가능한 테마

**애니메이션**:
- 카드 딜링 애니메이션
- 칩 베팅 애니메이션
- 승리/패배 이펙트
- 파티클 효과 (승리 시)

## 5. 기술 요구사항

### 5.1 성능
- 게임 로딩 시간 < 2초
- 애니메이션 60fps 유지
- 동시 접속자 10,000명 이상 지원
- 99.9% 업타임

### 5.2 보안
- HTTPS/SSL 암호화
- JWT 기반 인증
- SQL Injection 방지
- XSS/CSRF 방지
- DDoS 방어
- 데이터 암호화 (at rest, in transit)

### 5.3 공정성
- RNG(Random Number Generator) 인증
- 게임 결과 검증 시스템
- 독립적인 감사 (eCOGRA 등)
- 배당률(RTP) 공개

### 5.4 호환성
- 반응형 디자인 (데스크톱, 태블릿, 모바일)
- 크로스 브라우저 지원 (Chrome, Firefox, Safari, Edge)
- 다국어 지원
- 다중 화폐 지원

## 6. 비즈니스 모델

### 6.1 수익 모델
- 하우스 엣지(House Edge)를 통한 수익
- 게임별 RTP: 블랙잭 99.5%, 바카라 98.9%, 포커 레이크 2.5-5%

### 6.2 목표 시장
- 1차: 합법적인 온라인 도박 시장 (유럽, 일부 아시아 국가)
- 2차: 소셜 카지노 (가상 화폐 사용)

### 6.3 경쟁 우위
- 최고 품질의 그래픽 및 애니메이션
- 빠른 로딩 및 부드러운 게임플레이
- 사용자 친화적 인터페이스
- 투명한 운영 정책
